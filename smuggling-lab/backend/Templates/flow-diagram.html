Component Roles

HAProxy: Decides how to route requests based on Content-Length (in vulnerable mode).
Varnish: Caches responses. Vulnerable if it doesn't vary keys by user session.
Flask (Werkzeug): Processes requests. Vulnerable if it prioritizes Transfer-Encoding over Content-Length.

Scenario: CL.TE Desynchronization
Step 1: The Payload

Building the Malicious Request

An attacker sends a single POST request with dual framing headers. HAProxy (Front-end) uses CL to define request boundaries, while Flask (Back-end) uses TE.

POST / HTTP/1.1 Content-Length: 44 Transfer-Encoding: chunked 1\r\nX\r\n0\r\n\r\n GET /admin HTTP/1.1 # This is smuggled!
Step 2: Cache Poisoning

Hijacking the Victim's Cache Entry

When a victim requests /api/user, the backend processes the smuggled request instead. Varnish caches the sensitive admin response under the victim's URL.

Varnish Key: /api/user Cached Body: {"role": "admin", "secret": "XK9#mP2$"} Status: CACHE HIT (POISONED)
Other Vulnerabilities Demonstrated

Host Header Injection

Exploiting X-Forwarded-Host to hijack absolute URLs (Password Resets). If the backend trusts this header, it generates links pointing to the attacker's server.

X-Forwarded-Host: evil.attacker.com

Web Cache Deception

Appending /profile.css to an API path. Varnish caches private JSON data thinking it's a public static asset.

URL: /api/user/profile.css ‚Üí Result: Cached Private Data
Defense-in-Depth Strategy
Backend Defense

Strict Header Validation

Reject requests containing Transfer-Encoding or ambiguous framing. Flask app_secure.py now returns 400 Bad Request for any smuggling attempt.

Cache Defense

Key Isolation (Vary: Cookie)

Configuring Varnish to include session cookies in the hash key. This ensures that a poisoned entry for one user cannot be served to another.

Proxy Defense

HAProxy Normalization

Enforcing http-request deny if has_cl has_te to block the attack at the gateway before it reaches Varnish or the Backend.

Project Limitations & Research Scope

HTTP/1.1 Scope

The lab focuses on HTTP/1.1 framing issues. Modern HTTP/2 uses binary framing which naturally mitigates most Smuggling vectors.

Plaintext Traffic

All communication is over HTTP. While Smuggling works over TLS (HTTPS), the complexity of certificate handling was excluded from this lab.

Network Conditions

Simulated on localhost. Real-world attacks must account for network latency, TCP keep-alive timeouts, and load-balancer connection pooling.

Security Overhead

Implementing strict validation may block legitimate legacy clients. Production environments require a balance between security and compatibility.
